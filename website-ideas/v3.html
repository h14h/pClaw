The Problem With AI Agents

Most AI coding agents are built for companies with deep pockets. They assume you have GPU clusters, container orchestration, managed databases, and a team of platform engineers to babysit them. They require infrastructure that costs more per month than most freelancers earn. They are, in a word, bloated.

And yet the actual task—reading files, editing code, remembering context—doesn't require any of that. It requires an LLM, a few file operations, and something to remember what happened last Tuesday. That's it. The rest is organizational overhead disguised as engineering.

Worse still, most agents are stateless. Every conversation starts from zero. They don't know your codebase conventions, your preferences, your project history. You repeat yourself endlessly, spending tokens on context that should already be there. Each session is amnesia.

pClaw is the opposite. A single Go binary that runs on the cheapest VPS you can rent—1 vCPU, 1 GB of RAM, the $5/month tier. It talks to Vultr Inference at $0.20 per million tokens via Kimi K2 and GPT-OSS 120B. It has durable semantic memory backed by a vector store that builds over time. It remembers you.

No Docker. No Kubernetes. No Redis. No PostgreSQL. One environment variable, VULTR_API_KEY, and you're running. The entire toolset—file reads, file edits, directory listing, delegated reasoning, memory storage, memory recall, web search—is compiled into one binary with zero runtime dependencies.

This is radical simplicity. Not because simplicity is trendy, but because complexity is the enemy of accessibility. If your AI agent requires a DevOps team to deploy, it isn't a tool—it's a liability.

Capabilities

01

Read, Write, Edit

Full workspace file operations. Read any file, list directories, apply precise string replacements or create new files. No sandbox. Direct access.

02

Remember

Vector-backed semantic memory via Vultr's managed store. Your agent learns your codebase, your preferences, your patterns. Cross-session persistence that builds over time.

03

Reason

Delegates hard sub-problems to GPT-OSS 120B. The primary model knows when to call for backup. Bounded at 2 delegation calls per turn—enough to be useful, constrained enough to stay cheap.

04

Search

Web grounding via Tavily. The agent verifies factual claims before stating them. Facts, not hallucinations. Grounding rules are injected automatically when enabled.

05

Persona

Customizable identity system. Name it. Shape its behavior. Define its tone. The sectioned prompt builder composes identity, behavior, tooling, safety, and runtime sections into a coherent personality.

06

Connect

Terminal REPL today. Discord bot today. The transport layer is decoupled—HandleUserMessageProgressive drives any adapter. Everywhere tomorrow.

07

Compact

Rolling conversation summarization. When history exceeds 12KB, older messages are summarized and pruned while preserving tool call/result pair integrity. Never loses context.

Architecture

Component Purpose
Agent Loop Core chat/tool loop. Toggles between user input and model-tool-model continuation. Single-threaded with auxiliary goroutines for status rendering.
Inference Client Streaming and non-streaming paths to Vultr chat completions. Primary model: Kimi K2. Reasoning model: GPT-OSS 120B. Summarization: Qwen 2.5 Coder 32B.
Tool System Seven built-in tools registered at startup. JSON Schema input contracts. Async dispatch for fire-and-forget operations. Normalized event stream for lifecycle tracking.
File Tools read_file, list_files, edit_file. Direct filesystem access with no sandboxing. edit_file supports both replacement and creation paths.
Reasoning Delegation delegate_reasoning sends sub-problems to GPT-OSS 120B. Per-turn call limit of 2. No filesystem tools passed to the delegated model.
Memory System record and recall tools backed by Vultr vector store. Auto-recall injects summarized memories into every system prompt. Per-turn cache avoids redundant searches.
Web Search web_search tool via Tavily API. Supports general, news, and finance topics. Basic and advanced search depths. Grounding rules auto-injected into prompt.
Compaction Rolling summarization when conversation exceeds 12KB. Preserves tool call/result pair integrity at split boundaries. Non-fatal on failure.
Prompt Builder SectionedPromptBuilder composes Identity, Behavior, Tooling, Safety, Runtime, Memory, and Conversation Summary sections.
Discord Transport Per-(channel, user) session isolation. Progressive part callbacks. Message splitting with model-inserted markers and balanced chunking fallback.
