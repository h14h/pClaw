System Prompt Architecture

A sectioned prompt that composes identity, policy, and context into a single system message.

[Identity]
Name, role, model configuration
[Behavior]
Tone, interaction style, persona overlay
[Tooling]
Tool usage policy and constraints
[Safety]
Boundaries, refusal rules, guardrails
[Runtime]
Formatting directives, environment info
[Memory]
Auto-injected — recalled facts from vector store
[Conversation Summary]
Rolling compaction — injected when context exceeds threshold

Source Components

Six files. Each with a clear, single responsibility.

main.go
Agent runtime, inference client, tool definitions, REPL loop
  • Agent / Agent.Run()
  • runInferenceStream()
  • executeTool()
  • ToolDefinition
discord.go
Discord bot runtime and per-channel session management
  • DiscordBot
  • SessionManager
  • handleMessage()
memory.go
Durable semantic memory via vector-store backed recall
  • MemoryClient
  • Record() / Recall()
  • auto-recall injection
websearch.go
Web search grounding via Tavily API integration
  • WebSearchClient
  • Search()
  • web_search tool
prompting.go
System prompt builder with sectioned composition
  • BuildSystemPrompt()
  • section injection
  • persona merging
compaction.go
Rolling conversation summarization when context exceeds threshold
  • CompactConversation()
  • shouldCompact()
  • summary injection
